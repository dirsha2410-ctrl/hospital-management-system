# hospital-management-system
 	
1.Introduction

The Hospital Patient Management System is a software application designed to manage patient records and waiting processes in a hospital.
The main purpose of this system is to organize patient data, make searching easy, and handle patients efficiently.
In this project, data structures like Linked List and Queue are used to simulate real-life hospital operations.


 	2. Objective

To store patient data efficiently
To search patients using their ID
To arrange patients based on priority
To manage the patient waiting queue
To understand practical use of data structures


3. System Design / Algorithm

•	Add Patient

1. Input patient details
2. Create a new node
3. Insert it at the end of the linked list


Display Patients

1. Start from the head node
2. Traverse the linked list
3. Display each patient’s data

Search Patient
1. Input patient ID
2. Traverse the linked list
3. If ID matches, display patient details

 Sort Patients
1. Compare nodes in the linked list
2. Swap data based on priority
3. Display sorted list

 Queue Operations
Enqueue → Add patient ID to queue
Dequeue → Remove first patient from queue
Display → Show all patients in queue

 4. Implementation

The system is implemented using C++ programming language. Classes and pointers are used to create the linked list structure. The queue is implemented using arrays. 
A menu-driven approach is used for user interaction, allowing users to perform operations like adding, searching, displaying, and managing the queue.

5. Conclusion
This project successfully demonstrates a basic Hospital Patient Management System.
It helps in understanding the practical implementation of important data structures like Linked List and Queue.
The system efficiently manages patient data and simulates real-world hospital operations.
